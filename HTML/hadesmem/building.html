Building

HadesMem uses the Boost.Build build system. If you have not already, it is recommended you familiarize yourself with this framework.

Supported Compilers:

  • Microsoft Visual C++ 2010 (Express edition not officially supported)
  • GNU GCC 4.6.x (via MinGW-w64)
Warning

Extra scripts are present for other compilers, however these are currently unsupported and only present for use by developers and testers who wish to help 'port' the project to support these toolsets.

Supported Platforms:

  • Microsoft Windows (Windows XP SP2 or later)

Steps to Build:

  1. Build Boost.
    1. Navigate to Build/Boost.
    2. Run Bootstrap.bat.
    3. Run Build_Compiler_Architecture.bat (Substitute 'Compiler', and 'Architecture' as appropriate).
    4. Check output and ensure there are no errors.
  2. Build HadesMem.
    1. Navigate to Build/Full.
    2. Run Build_Compiler_Configuration_Architecture.bat (Substitute 'Compiler', 'Configuration', and 'Architecture' as appropriate).
    3. Check output and ensure there are no errors.
Caution

If using GCC, depending on which MinGW distribution you are using, you may need to manually configure toolsets in your user-config.jam file.

Using HadesMem (in a Boost.Build based project):

  1. Add a use-project directive to your Jamroot targeting HadesMem (this document assumes you name the project 'HadesMem').
  2. Add /HadesMem//HadesMem to the source list of your application/library or the requirements list of your project.
  3. Add #include <HadesMemory/Memory.hpp> to your code.
  4. Set the BOOST_ROOT environmental variable and copy b2 to somewhere in your PATH.
Important

When compiling your Boost.Build based project, explicitly specify the toolset, address-model and debug/release configuration on the command line to ensure that nothing needs to be recompiled and you are using the correct settings. If you've done it right then you should only need to link to Boost and HadesMem (i.e. nothing should be recompiled).

Sample Jamroot 

use-project /HadesMem 
  : 
    ../../HadesMem-Branches-v1.0.0/
  ;

exe Test
  : 
    Test.cpp
    /HadesMem//HadesMem
  ;

Sample Command-line: 

b2 toolset=msvc-10.0 address-model=32 debug
