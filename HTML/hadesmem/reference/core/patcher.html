Description

The PatchDetour class provides functionality for creating 'detours' in a remote process, and the PatchRaw class provides functionality for creating raw 'byte patches' in a remote process. Both classes inherit from the Patch base class. Please note that the Patch class is pure virtual and should not be used directly, only as a generic base.

PatchDetour uses a disassembler to automatically generate a trampoline, and also provides basic support for relative instruction resolution (primarily to allow hook chains, though other relative instruction types are supported).

Unlike some other hooking libraries, PatchDetour does not mix code and data, so targets are less likely to crash when hook chains are performed (either by your own code, or some third party code such as Steam or Fraps).

Warning

The destructors of patching objects do not remove the patch. This must be done either explicitly, or via the use of a RAII wrapper over the object.
