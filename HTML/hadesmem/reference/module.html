Description

The Module class provides functionality for finding and working with modules (a.k.a DLLs or EXEs).

The GetRemoteModuleHandle free function provides an 'extension' of the GetModuleHandle Windows API to allow lookup in a remote process.

The ModuleList class provides STL-compatible iterators for module enumeration. Module enumeration is 'lazy' to keep the API performant. Lazy in this context means that module lookup and initialization is only performed on-demmand, rather than the list being pre-processed. Furthermore, caching is implemented to improve performance when the list is used in multi-pass algorithms.
